## 这是什么？

你有没有想过：**学一门新手艺，最痛苦的不是「学不会」，而是「不知道该学什么」。**

网上关于任何领域的信息都是碎片化的——这个博主说要先学A，那个课程说B更重要，评论区又在吵C才是王道。你花了3个月，发现自己一直在绕弯路。

**鬼谷子.skill** 解决的就是这个问题。

它是一个 [Claude Code Skill](https://docs.anthropic.com/en/docs/claude-code)，输入任何一个领域名（比如「短视频」「独立开发」「咖啡拉花」），自动完成：

1. **6个维度并行调研**——经典著作、实战案例、社区共识、工具链、变现路径、学习路径
2. **三重验证提炼**——跨源复现 × 可操作性 × 反直觉性，过滤掉废话和正确的废话
3. **生成完整方法论**——核心原理、致命陷阱、阶段路线图、变现地图、诚实边界

输出的不是「学习笔记」，是**该领域的认知操作系统**。

---

## 为什么不用普通AI？

| 普通AI聊天 | 鬼谷子.skill |
|-----------|-------------|
| 单轮对话，想到什么说什么 | 6个Agent并行调研，覆盖6个维度 |
| 回答基于训练数据 | 实时搜索 + 多源交叉验证 |
| 输出笼统建议（「要坚持」「因人而异」） | 输出具体步骤（「每天做X，持续Y周」） |
| 没有质量保证 | 4轮质量测试 + 双Agent对抗打磨 |
| 一次性回复 | 生成完整文件系统，可持续迭代 |

**核心差异：鬼谷子用6个并行Agent做调研，这是网页AI做不到的。**

### SKILL.md 包含什么

- **领域画像**：一句话定义、适合谁、不适合谁、残酷真相
- **核心原理**（3-7条）：每条经过三重验证，附证据和具体用法
- **致命陷阱**（5-10条）：从真实失败案例提取，不是猜的
- **决策启发式**：遇到选择时的快速判断规则
- **阶段路线图**：存活期→成长期→变现期→规模化，每阶段有毕业标准
- **变现地图**：收入中位数、启动条件、从零到第一笔收入的步骤
- **资源地图**：必读（免费）、值得付费、不推荐（避坑）
- **领域争议**：有分歧的观点，不做判断，列出适用条件
- **诚实边界**：信息截止日期、低置信度部分、不能替代的东西

## 原理

### 流程架构

```
Phase 0: 入口路由
    ├── 完整蒸馏（新领域）
    ├── 快速补充（已有调研）
    └── 质量检测（已有Skill）
         ↓
Phase 1: 六维并行调研（6个Agent同时工作）
    ├── Agent 1: 经典著作
    ├── Agent 2: 实战案例
    ├── Agent 3: 社区共识
    ├── Agent 4: 工具链
    ├── Agent 5: 变现路径
    └── Agent 6: 学习路径
         ↓
Phase 1.5: 调研质量矩阵检查
         ↓
Phase 2: 三重验证提炼
    ├── 跨源复现（≥2个独立信源）
    ├── 可操作性（能转化为「当X时，做Y」）
    └── 反直觉性（新手听了会说「啊？真的吗？」）
         ↓
Phase 2.5: 框架确认（用户确认方向）
         ↓
Phase 3: Skill构建（按模板生成）
         ↓
Phase 4: 四重质量测试
    ├── 新手测试：新手读了能直接行动吗？
    ├── 深水测试：行业老手会觉得有价值吗？
    ├── 反废话测试：删掉领域名，还能和其他领域区分吗？
    └── 时效测试：数据有标注来源和时间吗？
         ↓
Phase 5: 双Agent对抗打磨
    ├── Agent A: 红队——找问题
    └── Agent B: 蓝队——改进
    └── 8维评分（≥7分才交付）
```

### 三重验证——为什么大部分方法论课是废话

市面上90%的方法论内容有一个共同问题：**正确但无用**。

「要深入理解用户需求」——对，但这是废话。
「要坚持不懈」——对，但这也是废话。

鬼谷子的三重验证专门过滤这些：

1. **跨源复现**：只有一个人说的不算数，至少2个独立信源验证
2. **可操作性**：必须能变成「当X时，做Y」的格式，不能是笼统建议
3. **反直觉性**：新手听了如果说「这不是显然的吗」，那就不够格

没通过三重验证的观点会被降级为「社区观点」或「领域争议」，不会进入核心原理。

### 诚实边界——鬼谷子知道自己不知道什么

每个生成的Skill都会标注：
- 信息截止日期
- 哪些结论置信度较低
- 哪些东西本Skill不能替代（需要亲身实践/导师指导的部分）
- 建议多久后重新蒸馏

**不装全知全能，该说不知道就说不知道。**

## 灵感来源

鬼谷子.skill 受 [女娲.skill](https://github.com/alchaincyf/nuwa-skill) 启发。

女娲蒸馏**人**——把一个人的思维方式和表达DNA提取出来。
鬼谷子蒸馏**手艺**——把一个领域的集体智慧提取成可执行的方法论。

两者互补：女娲告诉你「这个人怎么想」，鬼谷子告诉你「这门手艺怎么练」。

| 维度 | 女娲.skill | 鬼谷子.skill |
|------|-----------|-------------|
| 蒸馏对象 | 一个人的思维方式 | 一个领域的集体智慧 |
| 数据源 | 个人著作、访谈、社交媒体 | 多人经验、数据报告、社区共识 |
| 表达风格 | 用该人物的语气 | 中性直接，像经验丰富的前辈 |
| 输出格式 | 认知框架 + 表达DNA | 方法论 + 路线图 + 变现地图 |

## 质量检测

鬼谷子内置了质量检测机制，也可以手动运行：

```bash
# 检测生成的Skill
python3 scripts/quality_check.py path/to/SKILL.md

# 检测调研数据
python3 scripts/quality_check.py path/to/research/

# 合并调研数据
python3 scripts/merge_research.py path/to/research/
```

检测维度：
- 核心原理数量（3-7条）
- 致命陷阱数量（5-10条）
- 阶段路线图（≥3个阶段）
- 变现路径完整度
- 诚实边界（≥3条）
- 证据标注（≥5处）
- 废话检测（自动识别空洞表述）

## 适合蒸馏什么领域？

几乎任何有足够公开信息的领域都可以蒸馏：

**创作类**：短视频运营、播客制作、技术写作、自媒体
**技术类**：独立开发、数据分析、AI应用开发、爬虫
**商业类**：跨境电商、知识付费、社群运营、私域流量
**技能类**：咖啡拉花、摄影、健身、烹饪、演讲
**投资类**：量化交易、价值投资、加密货币

不适合蒸馏的：信息极度匮乏的领域（公开资料<2个独立信源）、纯学术理论研究、需要执照认证的专业领域（医疗、法律——鬼谷子会诚实告知）。

## 常见问题

**Q: 和直接问ChatGPT/Claude有什么区别？**
A: 普通对话是单轮生成，鬼谷子是6个Agent并行调研 + 三重验证 + 双Agent对抗打磨。信息密度和可靠性完全不同。

**Q: 生成一个领域要多久？**
A: 完整流程约10-20分钟（取决于领域复杂度和网络速度）。大部分时间在Phase 1的并行调研。

**Q: 生成的内容准确吗？**
A: 每条核心原理都经过三重验证，但鬼谷子会在「诚实边界」中标注低置信度的部分。没有哪个AI能100%准确——关键是鬼谷子会告诉你它不确定什么。

**Q: 可以二次编辑吗？**
A: 当然。生成的SKILL.md就是一个Markdown文件，你可以根据自己的经验修改、补充。
