# 领域方法论提炼框架

## 一、核心原理识别的三重验证

### 验证 1：跨源复现
同一个原理在至少2个独立信源中出现。

**合格示例**：
- 短视频领域：「前3秒决定生死」同时出现在抖音官方创作者手册、3个百万粉博主的复盘文章、以及平台数据报告中
- 独立开发：「先卖后造」同时出现在 Pieter Levels 的访谈、IndieHackers 社区高赞帖、以及 Y Combinator 的建议中

**不合格示例**：
- 只在一个人的课程里提到，没有其他人验证过
- 只在营销文案中出现，实际从业者没有提及

### 验证 2：可操作性
原理必须能转化为「当X时，做Y」的格式。

**合格**：「当你不确定选什么选题时，去评论区找被问得最多的问题」
**不合格**：「要深入理解用户需求」

### 验证 3：反直觉性
原理必须不是该领域所有人都知道的常识。

**测试方法**：把这条原理说给一个该领域的新手听，如果他的反应是「这不是显然的吗」，那就不够格。如果反应是「啊？真的吗？」或「我之前完全没想到」，那就是有价值的洞察。

### 未通过验证的观点处理
- 只通过1项验证 → 降级为「决策启发式」或「社区观点」，标注低置信度
- 一项都没通过 → 不收录
- 有争议（一些信源支持，另一些反对）→ 记录为「领域争议」，不做判断

## 二、致命陷阱的提取方法

### 来源优先级
1. **失败案例复盘**（最高价值）：从业者公开分享的失败经历
2. **社区高频吐槽**：新手区反复出现的相同问题
3. **专家警告**：行业老手反复强调的注意事项
4. **数据反模式**：和主流做法相反但数据证明有效的做法（反过来说，主流做法可能就是陷阱）

### 陷阱记录格式
每个陷阱必须包含：
- **具体行为描述**（不是笼统的「不要犯错」）
- **为什么新手会这么做**（理解错误的根源）
- **真实后果**（具体案例，不是假设）
- **正确替代方案**（具体到可以直接执行）

### 陷阱 vs 建议 的区分
- 「不要一开始就追求完美」→ 太笼统，不算陷阱
- 「不要花3个月打磨产品再上线，因为独立开发者 A 这样做了，上线后发现没人需要这个功能，浪费了3个月时间。正确做法：先做一个可以用的最小版本，7天内上线验证」→ 这是合格的陷阱

## 三、变现路径的真实性验证

### 收入数据标准
- **必须有来源**：从业者公开分享的真实数据，或行业报告的统计数据
- **标注范围**：给中位数，不给平均数（平均数被头部拉高，误导性极强）
- **标注前提条件**：这个收入水平需要什么前提（粉丝量、技能水平、时间投入）
- **标注时效**：这个数据是什么时候的，市场可能已经变化

### 收入数据红旗
以下情况必须标注警告：
- 只有头部案例的收入数据，没有中位数
- 数据来自培训课程的宣传页面
- 收入数据没有标注投入（收入10万但投入了8万，利润只有2万）
- 数据超过12个月未更新

### 「从零到第一笔收入」路径
每个变现方式都要提供：
1. 最小可行起点（需要什么技能/工具/资金）
2. 第一步做什么（具体到今天就能开始）
3. 预计多久能看到第一笔收入（给范围，不给精确数字）
4. 第一笔收入大概是多少（管理预期）

## 四、信息不足时的处理

### 信息密度分级

**充足**（≥10个独立信源）：可以提炼高置信度的方法论
**适中**（5-9个独立信源）：可以提炼，但标注部分结论为「中等置信度」
**稀少**（2-4个独立信源）：可以提炼框架，但大量标注为「低置信度」
**极度匮乏**（<2个独立信源）：不建议蒸馏，诚实告知用户

### 不同情况的策略

**领域太新**：标注为「前沿领域，方法论尚在形成中」，给出当前已知的最佳实践，明确说「3-6个月后建议重新蒸馏」

**领域太小众**：缩小范围，找到可以蒸馏的子领域。比如「水下摄影」信息太少，可以先蒸馏「摄影」的通用方法论，再补充水下的特殊要点。

**信息被商业化垄断**：很多领域的核心知识被包装成高价课程，公开信息少。标注为「该领域公开信息有限，以下方法论可能不完整」，同时列出可以获取更深入信息的途径。

## 五、领域Skill vs 人物Skill的区别

| 维度 | 人物Skill（女娲） | 领域Skill（鬼谷子） |
|------|------------------|-------------------|
| 蒸馏对象 | 一个人的思维方式 | 一个领域的集体智慧 |
| 数据源 | 个人著作、访谈、社交媒体 | 多人经验、数据报告、社区共识 |
| 表达风格 | 用该人物的语气 | 中性、直接、像经验丰富的前辈 |
| 矛盾处理 | 保留人格矛盾 | 记录流派分歧 |
| 输出格式 | 认知框架 + 表达DNA | 方法论 + 路线图 + 变现地图 |
| 使用场景 | 用某人的视角分析问题 | 从零学一门手艺并变现 |

## 六、质量自检清单

### 核心原理
- [ ] 每条原理都通过了三重验证？
- [ ] 每条原理都有至少2个独立信源的支持？
- [ ] 删掉领域名后，能和其他领域区分开？（如果不能，说明太泛了）
- [ ] 新手读了能说「原来如此」而不是「这不是废话吗」？

### 致命陷阱
- [ ] 每个陷阱都有真实案例？
- [ ] 每个陷阱都有具体的正确替代方案？
- [ ] 涵盖了新手最常犯的前5个错误？

### 路线图
- [ ] 每个阶段的时间预期是基于真实数据的？
- [ ] 每个阶段有明确的「毕业标准」？
- [ ] 刻意练习方案够具体？（不是「多练」，而是「每天做X，持续Y周」）

### 变现路径
- [ ] 收入数据有来源？
- [ ] 包含「从零到第一笔收入」的具体步骤？
- [ ] 有诚实的天花板和风险说明？

### 诚实边界
- [ ] 标注了信息截止日期？
- [ ] 标注了信息不足的领域？
- [ ] 标注了本Skill不能替代的东西？
- [ ] 给出了建议的重新蒸馏时间？

### 终极检验
**把这个Skill给一个该领域的新手看，问：「如果3个月前有人给你这份指南，你的起步会不会快很多？」**
如果答案是 YES → 通过。
如果答案是 不确定/没什么用 → 回 Phase 2 重新提炼。
